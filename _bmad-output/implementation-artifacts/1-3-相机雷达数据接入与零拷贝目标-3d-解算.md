# Story 1.3: 相机雷达数据接入与零拷贝目标 3D 解算

Status: in-progress

## Story

As a 机器狗系统,
I want 通过同进程 IPC 混图与雷达点云进行 3D 坐标解算,
so that 在资源受限时安全运行，不 OOM。

## Acceptance Criteria

1. Given 图像和点云流，When 对齐后解算，Then 在 < 50ms 内完成准确 PnP，合并占用内存不超过 60%(8GB)（NFR-1, NFR-2）。
2. And 应用 `boost::circular_buffer`（或等价固定容量封装）与 `unique_ptr`/`ConstSharedPtr` 零拷贝语义。

## Dev Notes

- 本故事是 Epic 1 的核心数据链路故事，目标是把现有“静态 TF + 雷达订阅骨架”升级为“图像+点云同步 + 3D 解算 + 零拷贝与定长缓存”。
- 仅做感知与解算链路，不引入运控动作调度逻辑（该部分归 Epic 2）。
- 结果消息需与 `dog_interfaces` 既有契约保持一致性，避免在 `dog_perception` 内定义私有通信类型。
- 所有参数与阈值外置化，保持赛场调参不需重编译。

### Project Structure Notes

- 重点改动范围（最小集合）：
  - `src/dog_perception/include/dog_perception/perception_node.hpp`
  - `src/dog_perception/src/perception_node.cpp`
  - `src/dog_perception/CMakeLists.txt`
  - `src/dog_perception/package.xml`
  - `src/dog_perception/test/test_perception_node.cpp`
  - `src/dog_interfaces/msg/*.msg`（若需补充 3D 结果字段）
- 结构约束：继续保持 `.hpp/.cpp` 分离；公共接口放 `include/<package>/`；日志统一 `RCLCPP_*`。

### Developer Guardrails

- 强制使用 ROS 2 IPC 友好的消息传递方式（发布 `unique_ptr`，订阅 `ConstSharedPtr`）。
- 强制使用固定容量环形缓存，不得出现不受控历史堆积。
- 消息契约优先复用 `dog_interfaces/msg/Target3D.msg`，不得在 `dog_perception` 内新建私有结果消息绕过接口防腐层。
- 禁止阻塞式同步等待；动作必须异步事件驱动。
- 禁止在高频回调中执行参数查询与磁盘 I/O。
- 坐标系命名必须遵循标准 TF 约定（`base_link`、`camera_optical_frame` 等）。

### Latest Tech Information

- ROS 2 Humble `message_filters` 文档建议：多源同步优先使用策略同步器（`ExactTime`/`ApproximateTime`），并确保订阅 QoS 与发布端一致，否则可能“无报错不回调”。
- ROS 2 QoS 文档强调传感器流通常采用 `SensorDataQoS`（更关注实时性），并需显式考虑兼容矩阵（reliability/durability 不匹配会断连）。
- `boost::circular_buffer` 文档强调其固定容量与覆盖写入语义，适合受限内存场景的滑窗缓存；容量应在初始化时明确并受配置约束。

### Library & Framework Requirements

- 运行基线：Ubuntu 22.04 + ROS 2 Humble + C++17。
- 必要组件：`rclcpp`、`sensor_msgs`、`message_filters`、`tf2`/`tf2_ros`、`yaml-cpp`、`Boost::circular_buffer`。
- 可选组件（按实现决定）：`cv_bridge`、`OpenCV`、`image_transport`。

### Testing Requirements

- 功能门禁：图像 + 点云同步后能稳定触发 3D 解算链路。
- 可靠性门禁：丢帧/旧帧可被丢弃且系统不中断；缓存不会无限增长。
- 质量门禁：`-Wall -Wextra` 零新增 warning；关键路径单测可重复执行。
- 性能门禁：在目标环境压测记录中，核心链路单帧处理保持 < 50ms 目标。
- 资源门禁：连续 30 分钟运行期间，`dog_perception` 进程内存占用曲线受控，峰值不超过系统 60%(8GB) 上限并具备日志证据。
