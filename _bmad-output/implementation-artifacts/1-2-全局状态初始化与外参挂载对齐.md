# Story 1.2: 全局状态初始化与外参挂载对齐

Status: done

## Story

As a 机器狗系统,
I want 在开机时从 YAML 热加载硬件坐标树并发布静态 `tf2`,
so that 发生形变时无需重编 CPP 即可矫正地图。

## Acceptance Criteria

## Dev Notes

- 本故事目标是“初始化对齐 + 配置外置化”，不实现视觉识别、PnP 解算或抓取逻辑。
- 必须遵循“参数不写死”约束：外参仅来自 YAML/参数，不允许硬编码到 `.cpp`。
- 当前骨架中 `dog_perception` 已有 `lidar_topic` 参数化订阅；本故事继续沿用“初始化读取参数 + 回调仅处理数据”的模式。
- 直接在现有 `PerceptionNode` 上增量扩展，不新增并行感知节点，避免重复订阅与并发路径分叉。
- 日志必须统一使用 `RCLCPP_*`，禁止 `std::cout/printf`。

### Developer Guardrails

- 禁止在高频回调中做参数拉取、YAML 文件 I/O、阻塞等待。
- 禁止使用 `std::this_thread::sleep_for()` 人为等待 TF 生效。
- 若外参加载失败，必须快速失败并可观测，不得“静默启动”。
- 仅发布静态 TF，不引入动态 TF 线程与额外执行器复杂度。
