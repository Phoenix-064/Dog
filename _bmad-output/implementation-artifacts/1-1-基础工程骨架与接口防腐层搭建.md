## Dev Notes

- 本故事目标是“骨架与契约先行”，不实现完整业务算法；避免在本 Story 引入与抓取逻辑相关的深层状态机。
- 现有仓库并存模块为 `livox_ros_driver2`、`point_lio` 与新增 `dog_*` 包；本故事改动不应破坏第三方模块构建入口。
- `dog_interfaces` 是唯一跨模块通信契约来源，后续包仅依赖该契约层，不做反向耦合。
- 强制零拷贝/智能指针风格为后续故事预留：回调优先 `ConstSharedPtr`，发布侧可迁移至 `unique_ptr`。
- 禁止在回调中使用 `std::this_thread::sleep_for()`，禁止 `std::cout/printf`，统一 `RCLCPP_*` 日志。

### Project Structure Notes

- 目录规划（新增）：
  - `src/dog_interfaces/{msg,srv,action,CMakeLists.txt,package.xml}`
  - `src/dog_perception/{include/dog_perception,src,CMakeLists.txt,package.xml}`
  - `src/dog_behavior/{include/dog_behavior,src,CMakeLists.txt,package.xml}`
  - `src/dog_lifecycle/{include/dog_lifecycle,src,CMakeLists.txt,package.xml}`
- 命名约束：包名全小写下划线；类名大驼峰；私有成员后缀 `_`；Topic/Action/Service 使用全小写路径。
- 结构约束：`.hpp/.cpp` 严格分离；公共头仅放 `include/<pkg>/`；禁止在 `.hpp` 写复杂内联实现。

### Technical Guardrails

- ROS 2 版本基线：Humble（Ubuntu 22.04），构建系统 `ament_cmake`。
- 保持接口与算法解耦：后续算法类必须可通过工厂创建（`std::unique_ptr<Interface>`），但 Node 本身不强制工厂。
- 参数与阈值不写死：配置后续统一放 `config/*.yaml`，本故事仅建立约定与占位。
- 若需引入 OpenCV/PCL/第三方库，必须同步声明并通过构建验证。

### Testing Requirements

- 编译质量门禁：`-Wall -Wextra` 零 warning（NFR-8）。
- 接口生成门禁：`.msg/.srv/.action` 可由构建系统正确生成并被其他包引用。
- 最小回归门禁：新增包不影响现有 `livox_ros_driver2` 与 `point_lio` 的既有源码结构。
